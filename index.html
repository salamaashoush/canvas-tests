Target Bug Configuration

The reported bug occurs on:

  • OS: Windows
  • Browser: Firefox 145, 146
  • Issue: Blank/empty signatures being submitted
  • Symptoms:
    • Image dimensions preserved but data URI varies in length
    • For text placeholders: height can be 1px or 5px (incorrect)
    • Canvas appears to be blanked or filled with transparent pixels

How to Use These Tests

  1. Run each test in Firefox on Windows to attempt reproduction
  2. Compare results with Chrome/Safari on macOS to see differences
  3. Look for FAIL or WARNING results
  4. Check the browser console for additional errors
  5. Document any differences between browsers
Tip: Try enabling Firefox's fingerprint resistance (privacy.resistFingerprinting) to test if privacy settings affect canvas behavior.

Test Files

1 Font Metrics actualBoundingBox Can Be Zero

Tests if actualBoundingBoxRight/Left/Ascent/Descent can return 0, causing canvas dimensions to become 0.
2 Font Loading Race Condition

Tests if measuring text before fonts load produces different/incorrect metrics.
3 Canvas Clear Behavior on Resize

Tests if canvas properly clears when dimensions change, and if context state persists.
4 Font Size Loop Can Reach Zero

Tests the font-sizing algorithm from the text signature module to see if fontSize can become 0.
5 toDataURL Timing Issues

Tests for timing issues between drawing operations and toDataURL() capture.
6 Firefox-Specific Canvas Issues

Tests Firefox-specific behaviors: DPR, precision (f32 vs f64), fingerprint resistance, GPU acceleration.
7 Complete Signature Flow Simulation

Interactive test simulating the complete signature capture flow with draw and text modes.
8 Edge Cases and Failure Modes

Tests context loss, willReadFrequently, canvas size limits, transform state, privacy protection, cross-origin tainting, GPU acceleration.
9 Real PDF Loading with PDF.js

Load and render real PDF files using PDF.js library, then generate signatures under memory pressure. Mimics exact customer experience with large documents.
